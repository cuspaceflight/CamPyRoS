Want to help out?¶

We don’t have any formal contribution guidlines, code styles or practices yet but any help would be greatly appreciated. If you see a problem that you don’t know how to fix please make an issue or if you fix one please make a pull request and well see what you can do. If you have any questions please email too.

Style pointers¶

  • Vaguly follow normal convtions (PEP8)

  • Use docstrings on functions, these must be surrounded by “”” (not ‘’’) and should be google format (most are numpy but we are transitioning away)

  • Comments not for documentation should use # (i.e. pointers about what individual code sections do)

  • Docstrings are whitespace sensitive

  • Please keep a note in readme or elsewear of papers you have used
